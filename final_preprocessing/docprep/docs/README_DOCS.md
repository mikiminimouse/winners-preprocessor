## Ключевые концепции

### Подход к детекции файлов

**Используем стандартный каскадный подход из production-систем:**

```
1. python-magic (magic bytes) → базовая детекция
2. Структурный парсинг (zipfile, pypdf) → для полиглотных форматов
3. Валидация (testzip) → проверка целостности
4. Сравнение (fake_doc) → обнаружение обмана расширений
```

**Это не кастомная логика, а стандартная практика!**

### Архитектура системы

```
Input/ → Classifier → Pending_N/ → Converter/Extractor/Normalizer → Merge_N/ → Ready2Docling/
```

Все компоненты интегрированы с файловой системой и автоматически:
- Перемещают UNIT между директориями
- Обновляют state machine
- Создают и обновляют manifest
- Логируют в audit log
